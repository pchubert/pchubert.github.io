两数相加

发表于 2019-03-11 | 阅读次数:

  逐渐走入周更的歧途，以及在码代码上再次感受落差，虽然也有一部分外因，啊，废话不多说，接着写吧。

题目：

  给出两个非空的链表用来表示两个非负的整数。其中，它们各自的位数是按照逆序的方式存储的，并且它们的每个节点只能存储一位数字。如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。
  您可以假设除了数字 0 之外，这两个数都不会以 0 开头。
示例：
输入：(2 -> 4 -> 3) + (5 -> 6 -> 4)
输出：7 -> 0 -> 8
原因：342 + 465 = 807

难度：中等
分析：最开始甚至想过先还原L1,L2链表所代表的数，然后重建新的链表，然而实际上这样是低效的，本身处理构建目标链表的难度不变不说，之前还加入了解析两给定链表的需求。后面转念了，直接在链表上考虑，但当时没想到以最长为目标，而取了以最短为循环的跳出基准，这点就又想歪了，废了不少功夫，觉得实在麻烦，然后不得已看了官方解答。
  其实好好思考下其实不难，既然可能出现某个链表较短的情况，那就让之后它所代表的每个数字都为0就好了，一直加到较长链的链尾，最后再考虑一下最高位是否有进位，并新建节点即可。官方题解里的特殊情况是有点问题的，不知道是不是翻译的原因，都指明了非空链表了，是不可能出现空链表的，但题解里有指明，我也比较较真，也就考虑进去了（指允许链表空）。

暴力破解法

  链表正常操作，取q,p为两原ListNode指针的程序当前指针，它们随循环前进，carry为进位，只要q,p一方的next不为NULL就让循环继续，只要一方为NULL，那么它此轮循环以及之后的val就代为0参与sum计算，实际存入目标链表当前节点的值为sum的个位部分，而进位carry则根据sum的十位部分取。因为确保了某一方的next有值，所以也就放心new ListNode(0)了，注意这里是有设计好的构造函数即可。
  循环次数其实在这份代码的次数为max(m,n)-1，时间复杂度概括来讲也就是O(max(m,n))，而新列表最多为max(m,n)+1，空间复杂度笼统来讲也就是O(max(m,n))。
