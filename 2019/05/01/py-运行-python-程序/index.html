./*.py 运行 python 程序

发表于 2019-05-01 | 阅读次数:

  刚入门，也是慢慢填坑，毕竟想往信息安全那边靠，虽然做不到说看完书就学会，但就当实验做做，权当扩展知识面。这一篇只是为了想像书本一样简洁地敲命令行而记录的。

内容

首先看下正常情况下是不可以直接执行 .py 文件的

我们就需要两步，但也不能完全做到永续性简便就是了，第一步在要运行的 .py 文件首行加入”#!/usr/bin/python”（具体路径可以看下自己的 python 路径在哪，”#coding=utf-8”有编码需求可以上这个）

第二步，打开终端，直接用 chmod 命令，如”chmod 755 ./*.py”，更改 .py 文件的运行权限，终端输入”./*.py”发现运行成功

对比一下不经过处理的运行方式，”python ./*.py”，也许现在看不出什么，但好处就是命令更简洁了，而且对于多次测试 python 程序，只需要一次更改就可以了，嗯，还是有好处的，见仁见智吧
